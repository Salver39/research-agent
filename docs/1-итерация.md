## Pipeline — 9 шагов

| Шаг | Стадия | Что происходит |
|-----|--------|----------------|
| 1/9 | intake | Пользователь описывает задачу; GPT-5.4-mini генерирует 3 уточняющих вопроса |
| 2/9 | clarify | Пользователь отвечает на вопросы; GPT-5.5 формулирует структурированный бриф |
| 3/9 | brief | Пользователь правит и подтверждает бриф (decision — обязательное поле) |
| 4/9 | context | Drag & drop загрузка файлов (.pdf, .docx, .txt, .csv); RAG-индексация в Chroma; ContextAgent анализирует паттерны |
| 5/9 | hypothesis | HypothesisAgent формулирует 3–5 фальсифицируемых гипотез; пользователь может добавить свои или удалить лишние |
| 6/9 | method | MethodAgent выбирает метод по матрице неопределённости с обоснованием |
| 7/9 | sampling | SamplingAgent определяет сегменты, размер выборки, критерии и скринер-вопросы |
| 8/9 | design | DesignAgent собирает Discussion Guide по блокам; каждый блок закрывает конкретную гипотезу |
| 9/9 | done | Генерация 6 документов; скачивание по одному или ZIP-архивом |

## Агенты — формат вывода

Все агенты возвращают строгий JSON (без markdown-блоков). Промпты требуют этого явно.

| Агент | JSON-формат |
|-------|-------------|
| BriefAgent (intake) | `{"questions": ["?", "?", "?"]}` |
| BriefAgent (clarify) | `{"research_question": "...", "decision": "...", "constraints": "...", "known": "..."}` |
| ContextAgent | `{"patterns": [{"text": "...", "source": "..."}], "summary": "..."}` |
| HypothesisAgent | `{"hypotheses": [{"id": "h1", "text": "...", "source": "...", "priority": 1, "falsifiable": true}]}` |
| MethodAgent | `{"name": "...", "uncertainty_type": "...", "rationale": "...", "participants": 8, "duration": "...", "format": "..."}` |
| SamplingAgent | `{"segments": [...], "total_size": 8, "criteria": {...}, "screener": [...]}` |
| DesignAgent | `{"guide_blocks": [...], "total_duration": "60 мин", "quality_checks": {...}}` |

---

## Документы на выходе

| Документ | Содержит |
|----------|----------|
| Discussion Guide | блоки из `design.guide_blocks` с вопросами и целями |
| Рекрутинговый скринер | сегменты, критерии включения/исключения, скринер-вопросы |
| Briefing для интервьюера | гипотезы + правила нейтрального проведения |
| Шаблон инсайтов | секция на каждую гипотезу с полями для наблюдений и цитат |
| Дизайн исследования | бриф + метод + выборка + quality checks |
| Чеклист запуска | 3 раздела: подготовка, рекрутинг, проведение |

Доступны форматы: `.docx` и `.pdf` (через weasyprint). Скачивание ZIP-архива — одной кнопкой.

## Технические решения принятые в итерации

### LLM
- Начинали с Anthropic (Claude Sonnet) — не прошла оплата картой
- Переключились на OpenAI: GPT-5.5 для основных агентов, GPT-5.4-mini для уточняющих вопросов
- Параметр `max_completion_tokens` вместо `max_tokens` (требование GPT-5.5)
- SDK обновлён с 1.30.1 до 2.35.1

### SSE стриминг
- Backend: FastAPI `StreamingResponse` с генератором `async for chunk in agent.stream(...)`
- Frontend: нативный `fetch` + `ReadableStream.getReader()` — без EventSource (он не поддерживает POST)
- Состояние сохраняется в БД через `BackgroundTasks` после окончания стрима

### State machine
- Стадии: `intake → clarify → brief → context → hypothesis → method → sampling → design → done`
- `scenario` удалён — содержимое Discussion Guide уже включено в `DesignScreen`
- State сериализуется в SQLite после каждого этапа; сессию можно продолжить по URL

### Исправленные баги
- `max_tokens` не работает в GPT-5.5 → заменён на `max_completion_tokens`
- `load_dotenv()` не подхватывал `.env` → добавлен `override=True` и абсолютный путь
- Greenlet не был установлен → SQLAlchemy async падал при старте
- `next.config.ts` не поддерживается Next.js 14 → переименован в `next.config.mjs`
- Шаг 8 прыгал на 10 → удалён `scenario` из STAGES, исправлен `onDesignConfirm`
- `onContextContinue` делал advance перед стримингом → ContextAgent запускался на неправильном стейдже → убран лишний advance
- ZIP содержал дублирующиеся файлы → добавлен `seen` set при формировании архива
- DesignAgent JSON обрезался → увеличен `max_completion_tokens` до 6000

## UX-улучшения

### Экран гипотез
- Во время стриминга показывается **живой прогресс**: пульсирующий индикатор, прогресс-бар из 4 сегментов, счётчик «2 из ~5»
- Гипотезы появляются по одной с анимацией fade-in по мере того как JSON стримится
- Skeleton-карточки пока следующая гипотеза ещё набирается
- Кнопка «+ Добавить свою гипотезу» — разворачивается в форму с подсказкой формата
- Удаление гипотезы — кнопка `×` появляется при наведении
- `Cmd+Enter` для подтверждения, `Escape` для отмены
